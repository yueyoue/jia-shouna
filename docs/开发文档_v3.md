## 1. 项目概述

「家收纳」是一款面向家庭用户的物品全生命周期管理工具，解决家庭物品找不到、记不住、易过期、重复买的痛点。

### 1.1 核心功能

- **多级收纳空间**：家 → 房间 → 容器 三级层级管理
- **物品全生命周期**：录入、查找、领用、归还、过期提醒
- **多种录入方式**：条码扫描、拍照识别、AI智能识别、手动录入
- **家庭协作**：多成员共享、物品级隐私控制
- **离线支持**：无网络时可查看已缓存的物品数据
- **智能提醒**：临期物品、库存不足提醒

### 1.2 技术栈

| 端 | 技术 |
|---|---|
| APP端 | Android 原生 (Java, API 26+) |
| 后端 | PHP 7.4+ / 8.0+ |
| 数据库 | MySQL 5.7+ / 8.0+ |
| 管理后台 | PHP + HTML + CSS + JS |
| 构建 | GitHub Actions 自动编译 |

### 3.2 物品录入 (AddItemActivity)

**四种录入方式：**

| Tab | 功能 | 说明 |
|-----|------|------|
| 📷 扫码 | 条码扫描 | 调用ZXing扫码，自动查询条码信息 |
| 📸 拍照 | 拍照录入 | 拍照添加物品图片，不自动识别 |
| ✏️ 手动 | 手动录入 | 纯手动填写表单 |
| 🤖 AI识别 | AI智能识别 | 拍照后AI自动识别物品信息 |

**表单字段：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 存放位置 | 选择器 | ✅ | 三级空间选择 |
| 物品名称 | 文本 | ✅ | 物品名称 |
| 条形码 | 文本 | - | 扫码或手动输入 |
| 品牌 | 文本 | - | AI识别或手动输入 |
| 数量 | 数字 | - | 默认1 |
| 单位 | Spinner | - | 个/盒/瓶/包/袋/罐/箱/件/套 |
| 生产日期 | 日期 | - | 点击选择 |
| 保质期 | 数字+单位 | - | 天/月/年，自动计算过期日期 |
| 过期日期 | 自动计算 | - | 根据生产日期+保质期计算 |
| 购买价格 | 数字 | - | 可选 |
| 物品标签 | 多选 | - | 从已有标签选择或新建 |
| 物品照片 | 图片 | - | 拍照添加，支持删除 |
| 隐私设置 | 开关 | - | 仅自己可见 |

**AI识别流程：**
1. 点击"AI识别"Tab → 调起相机拍照
2. 图片上传到 `ai/recognize.php`
3. 返回识别结果（名称、品牌、条码、分类、保质期）
4. 用户确认后自动填入表单
5. 识别的照片自动添加到物品图片
6. 分类自动添加为标签

## 7. 离线功能

### 7.1 设计思路

采用**只读缓存**方案：
- 联网时自动缓存物品列表到本地SQLite
- 离线时从缓存加载，只读不可编辑
- 图片不缓存（只存URL），不占额外空间
- 缓存容量约1MB/1000件物品

### 7.2 本地数据库 (LocalDb)

**数据库名**：`jiashouna_local.db`

**缓存表**：`cache_goods`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 服务器物品ID |
| house_id | INTEGER | 家庭ID |
| space_id | INTEGER | 空间ID |
| name | TEXT | 物品名称 |
| barcode | TEXT | 条形码 |
| category | TEXT | 分类 |
| brand | TEXT | 品牌 |
| quantity | REAL | 数量 |
| unit | TEXT | 单位 |
| purchase_date | TEXT | 购买日期 |
| expiry_date | TEXT | 过期日期 |
| purchase_price | REAL | 价格 |
| note | TEXT | 备注 |
| is_private | INTEGER | 隐私 |
| space_name | TEXT | 空间名称 |
| space_icon | TEXT | 空间图标 |
| cover_image | TEXT | 封面图URL |
| cached_at | INTEGER | 缓存时间 |

**离线创建表**：`local_goods`（用于断网时创建的物品，联网后同步）

### 7.3 离线行为

| 功能 | 离线可用 | 说明 |
|------|---------|------|
| 查看物品列表 | ✅ | 从缓存加载，顶部显示"离线模式" |
| 查看物品详情 | ✅ | 从缓存加载，禁用编辑 |
| 搜索物品 | ✅ | 搜索本地缓存 |
| 首页最近物品 | ✅ | 从缓存显示最近4件 |
| 录入物品 | ❌ | 提示需要联网 |
| 编辑物品 | ❌ | 提示需要联网 |
| AI识别 | ❌ | 需要网络调API |

## 8. 日志收集系统

### 8.1 架构

```
APP端                    服务器                 管理后台
┌─────────┐  HTTP POST  ┌─────────┐   SQL    ┌──────────┐
│AppLogger│ ──────────► │ log.php │ ───────► │ app_log  │
│         │             │         │          │   表     │
└─────────┘             └─────────┘          └──────────┘
```

### 8.2 APP端 (AppLogger.java)

**初始化**：在 `App.onCreate()` 中调用 `AppLogger.init(this)`

**功能：**
- 全局未处理异常捕获（UncaughtExceptionHandler）
- crash日志同步上报（确保崩溃前数据发出）
- error/warn/info 日志缓冲上报（满5条自动上传）
- API错误记录

**日志数据：**
- type: crash/error/api/warn/info
- tag: 标签（如类名）
- message: 错误信息
- stack_trace: 堆栈跟踪
- device_info: 设备型号、Android版本
- app_version: APP版本号
- user_id: 用户ID

### 8.3 后端 (log.php)

**接口：**
- `POST ?action=upload` — 接收日志（支持单条和批量）
- `GET ?action=list` — 日志列表（支持分页、类型筛选）
- `GET ?action=stats` — 统计（总数、今日、崩溃数）
- `POST ?action=clear` — 清空日志

### 8.4 管理后台 (app-logs.php)

- 统计卡片：总日志、崩溃数、错误数、今日新增
- 日志列表：类型标签、消息、设备信息、时间
- 筛选：按类型（crash/error/api/warn/info）、关键词搜索
- 详情：点击展开查看完整堆栈
- 操作：清空日志

## 10. 构建与发布

### 10.1 GitHub Actions

**触发条件：**
- push到main分支 → 自动构建+发布Release
- 手动触发 (workflow_dispatch)

**构建流程：**
1. Checkout代码
2. JDK 17环境
3. 解码签名密钥 (KEYSTORE_BASE64 secret)
4. 编译Debug APK
5. 编译Release APK（签名）
6. 打包Web部署包
7. 生成版本号（v{日期}.{时间}）
8. 创建GitHub Release

**签名配置：**
- 使用固定的 `release.keystore.p12` 签名
- 所有版本签名一致，避免更新失败
- 签名信息存储在 GitHub Secret `KEYSTORE_BASE64`

**Release产物：**
- `app-debug.apk` — 调试版
- `app-release.apk` — 签名发布版
- `jia-shouna-web.tar.gz` — Web后端部署包
